# プロジェクト概要・スコープ・用語

## ゴール

AngryMiao **CyberBoard R4** の設定を、純正ソフト **AM Master** を使わずに
CLI から直接キーボードへ書き込めるツールを作る。具体的には:

1. **独自スキーマでキーマップを管理**(キーマップと LED を分離して個別編集)。
2. **LED ディスプレイのスロット 1/2/3 用アニメーション指定**(= page 5/6/7)。
3. それらを **CLI から直接書き込む**。

## なぜ作るのか(動機)

- 純正 JSON は**キーマップ + 各キー LED + LED ディスプレイ**が 1 ファイルに同居。
  コミュニティ製 LED 設定を適用すると**キーマップまで上書き**される。
- 純正アプリは**接続が不安定**(書ける/書けないのムラ)。
- 小回りが利かない。

→ 分離管理 + 堅牢な書き込み + 部分更新を CLI で実現する。

## スコープ

| 対象 | 状態 |
|---|---|
| CyberBoard R4(本体・有線) | 主対象 |
| ドングル(無線 `AM35_D`) | 視野に入れる(経路が別、`30` 参照) |
| R2/R3 など旧世代 | 互換性は将来検討 |
| ファーム更新(MediaTek `AM_TOOL`) | **対象外**(別系統) |
| オーディオ/EQ 機能 | 対象外 |

## 用語

| 用語 | 意味 |
|---|---|
| **AM Master** | 純正設定アプリ(PyInstaller/Python3.7/PySide2) |
| **hatsu line / page** | 発光ライン/ページ = 上部 40×5 LED マトリクス(ディスプレイ) |
| **スロット 1/2/3** | Custom LED 1/2/3 = `page_data` の index 5/6/7 |
| **key_layer** | レイヤ別キーリマップ(7層×200キー) |
| **frames / keyframes** | 確定フレーム(200px)/ 補間用キーフレーム(可変長) |
| **AM35 / AM35_D / CBR5** | アプリ内の製品コード(本体/ドングル/別製品) |
| **IR** | 自作スキーマと書き込みの間に挟む中間表現(= 純正 JSON。`40` 参照) |
| **merger** | 既存の別ツール `angrymiao-cyberboard-config-merger`(Python) |
| **miaomerge** | merger の Tauri(React+Rust)版リライト。LED マージ→JSON 保存のみ(書き込み無し)|

## ドキュメントの使い方

- 事実は**確度付き**で書く(🟢確認 / 🟡推定 / 🔴未確認)。
- 新しい発見は `90-research-log.md` に**時系列で追記** → 確定したら該当 rule に反映。
- 実機キャプチャで裏が取れるまで `30-write-protocol.md` は**仮説**として扱う。
- 原始的な調査資材(`_re/` の解析スクリプト・`experiments/` の実験 README/データ)は
  [**GitHub wiki**](https://github.com/GeneralD/cyberboard-cli/wiki) へ移設(`90` 続30)。
  本 rule 群が指す `experiments/...` パスは wiki / git 履歴を参照のこと。
